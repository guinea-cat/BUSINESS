# 提示词修改说明

## 1. 修改背景
针对「AI黑客松创新性&社会价值评估系统」进行核心提示词优化，旨在通过引入「真人专家分步评估工作流」，提升 Agent 的输出专业度、证据链完整性，并彻底解决模型幻觉问题。

## 2. 修改点及真人工作流来源

### A. 创新性评估模板 (innovation_report_optimizer)
| 修改模块 | 真人专家工作流来源 | 优化目标 |
| :--- | :--- | :--- |
| **技术栈核查步骤** | 来源：专家「生态成熟度核查」及「技术对标」步骤。 | 确保技术栈分析有据可查，识别前沿性。 |
| **技术原创性核查** | 来源：专家「自主代码比例核查」与「改进逻辑识别」。 | 区分套壳项目与自主创新。 |
| **同类项目对标** | 来源：专家「行业 Benchmark 对比」。 | 提供行业洞见，避免泛泛而谈。 |
| **评分模型复用** | 来源：专家「标准化评审效率手册」。 | 减少 30% 重复操作，提升评估一致性。 |

### B. 社会价值评估模板 (social_value_analyzer)
| 修改模块 | 真人专家工作流来源 | 优化目标 |
| :--- | :--- | :--- |
| **底线风险核查** | 来源：专家「伦理与合规审计」步骤。 | 快速识别红线风险，一次性完成 3 项检查。 |
| **受益群体画像** | 来源：专家「人文关怀度评估」手册。 | 深挖弱势群体适配细节，提升评估深度。 |
| **影响可量化性** | 来源：专家「社会影响力评估 (SIA)」标准。 | 强制提取可验证的社会效益数据。 |

## 3. 反幻觉与质量保障措施
1. **强制证据绑定**：要求所有结论必须附带 GitHub 文件路径及具体内容。参考央视新闻 AI 幻觉防控研究，此举可降低约 70% 的模型虚假信息编造风险。
2. **禁言模糊表述**：明确禁止使用“可能”、“潜力”等主观词汇，信息不足时强制标注 `【信息不足】`，确保评估报告的权威性。
3. **差异化对比**：在模板中新增「本系统 vs 通用大模型」对比模块，迫使模型从专家视角输出差异化洞见。

## 4. 运行时修复
- **PromptTemplate 参数修复**：解决了运行时 `examples` 参数缺失导致的初始化异常，确保系统稳定性。
